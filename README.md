# Технологический стек проекта

Веб-проект интернет-магазина сладостей реализован на стеке технологий **.NET**

##  Архитектура и Backend
- **ASP.NET Core MVC & Razor Pages**: Служит основой для построения структурированного, многослойного backend-приложения. MVC обеспечивает чёткое разделение ответственности (модель, представление, контроллер), в то время как Razor Pages предлагает более простую страничную модель для задач, ориентированных на отдельные страницы.
- **ASP.NET Core Web API**: Предоставляет независимый, RESTful слой для взаимодействия с клиентскими приложениями (включая Blazor WebAssembly). Это позволяет отделить бизнес-логику от представления и создать основу для потенциального развития мобильных или десктопных клиентов.
- **Entity Framework Core**: Мощный объектно-реляционный маппер (ORM), который упрощает работу с базой данных, позволяя взаимодействовать с ней через объекты C#. Это ускоряет разработку, повышает безопасность (защита от SQL-инъекций) и обеспечивает переносимость кода.

##  Frontend и Пользовательский интерфейс
- **Blazor (гибридный подход)**: Проект использует современную веб-технологию от Microsoft, которая позволяет писать интерактивный клиентский код на C# вместо JavaScript.
    - **Blazor WebAssembly**: Запускает код .NET прямо в браузере пользователя, создавая отзывчивый и плавный пользовательский интерфейс, характерный для одностраничных приложений (SPA), без постоянных перезагрузок страниц.
    - **Blazor SSR (Server-Side Rendering)**: Используется для страниц, где критически важна скорость первоначальной загрузки. Страница полностью формируется на сервере и отправляется браузеру в готовом виде, что положительно влияет на SEO и воспринимаемую производительность.
- **Razor Pages**: Используется для статичных или простых динамических страниц, обеспечивая быструю разработку с логикой, тесно связанной с представлением.

##  База данных
- **PostgreSQL**: объектно-реляционная система управления базами данных с открытым исходным кодом. Выбрана за её стабильность, богатый функционал, строгую соответствие стандартам SQL и эффективность при работе со сложными данными, что важно для каталога товаров, заказов и управления клиентами.

##  Безопасность и Аутентификация
- **Keycloak**: Внешний сервер управления идентификацией и доступом (IAM) с открытым исходным кодом. В проекте он отвечает за:
    - **Централизованную аутентификацию** (Single Sign-On - SSO).
    - **Безопасное управление пользователями, ролями и разрешениями**.
    - **Поддержку стандартных протоколов** (OpenID Connect, OAuth 2.0).
    - **Делегирование сложной логики безопасности** (сброс пароля, двухфакторная аутентификация, социальные логины) внешнему специализированному сервису, что повышает общую безопасность системы.

##  Качество и Эксплуатация
- **Тестирование (Tests)**: Реализована многоуровневая стратегия тестирования, включающая модульные и интеграционные тесты. Это обеспечивает стабильность работы ядра приложения, бизнес-логики и интеграции с базой данных, упрощает рефакторинг и снижает количество ошибок.
- **Логирование (Logs)**: Интегрирована система структурированного логирования для записи событий, предупреждений и ошибок приложения. Это ключевой инструмент для мониторинга работы системы в реальном времени, оперативного выявления проблем и их последующего анализа.
